Zenburn, I love you.. October, 2011

I am a vim user. Leo was my choice of a colorscheme. It offers quite a palette of well-balanced colours. Contrast is adequately high while on a laptop.

^LEO

When I switched to a desktop, I spent substantial time trying to figure out the perfect brightness/contrast ratio. One that soothed the eyes.

I don't recall how I came across you, zenburn, but my eyes are thankful that I did. You make my background a lighter shade of grey and use a subtle palette of pastel colours. The lowered contrast enables me to take code in for longer hours. What would I do without you!

^Zenburn

Refactoring Java, not again! Jun, 2011

Paul Graham very rightly says, a programming language should be malleable. It should allow you to quickly prototype a project and then beat the code into shape.

Static typing, like in java, is prohibitive of this approach. Another feature I feel is unnecessarily bureaucratic : interfaces. Interfaces sound too good to be true. They smell a lot like modularity and it is remarkable how if used correctly, they can lead to highly decoupled code. But interfaces also hide a lot of data. The actual implementation may contain substantial information that is inaccessible to outside code.

The more I program in Java, the more I feel it spends more time preventing me from doing things, than helping me out. In a way, you turn out to be your own "pointy-haired-boss". First, the PHB in you writes the interface. Then the employee in you starts writing an implementation based on the guidelines laid down by the PHB. Once this is done, you both are happy until there arises a requirement for some more features that this interface doesn't support. You send a request to PHB, he writes more of the interface and the employee in you writes more code to implement the interface and this cycle continues.

I am writing a RIA in GWT. I am trying to adhere to the design patterns laid out in the documentation. Sometimes, in the spirit of the code, just to make something work, I tend to step out of the design pattern and code arbitrarily. Now, when I have to refactor the code to align it to the pattern, I find myself terrified of the effort it would take to acheive that.

Having been on Java for about 3 months, when I try to write python code, I find myself thinking in Java. I find the lack of types and interfaces difficult to manage. I am writing up un-pythonic reflection based systems to implement features similar to interfaces and type-checking in java.

Python, I shall see you soon, and when I do...
